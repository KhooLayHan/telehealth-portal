# Architecture Decision Records (ADRs)

An Architectural Decision (AD) is a software design choice that addresses a functional or non-functional requirement that is architecturally significant.

An Architectural Decision Record (ADR) captures an important architectural decision made along with its context and consequences.

## Contributing

When creating a new ADR:

1. Copy the [template](0001-pulumi-infrastructure-deployment-strategy.md) (use the latest as reference)
2. Name the file `NNNN-title-with-dashes.md` where `NNNN` is the next number in sequence
3. Add the entry to the table above
4. Place the file in `docs/decisions/`
